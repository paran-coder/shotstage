# ShotStage 사용자 매뉴얼

ShotStage는 3D 뷰포트에서 카메라 앵글·렌즈·샷 타입을 정확히 잡고, 그 설정을 그대로 AI 이미지/영상 생성 프롬프트로 변환해주는 도구입니다. 앱 안에서는 상단의 **이용 방법** 버튼(`/guide`)으로도 같은 내용을 볼 수 있습니다.

---

## 1. 기본 흐름

1. 3D 뷰포트에서 마네킹(스탠드인 캐릭터)을 원하는 구도로 배치
2. 오른쪽 패널에서 샷 타입 · 카메라 · 피사체 · 프롬프트 내용 설정
3. **생성하기** 버튼 클릭
4. 결과 모달에서 블로킹 프레임 이미지 + 완성된 프롬프트 확인, 복사/다운로드
5. Higgsfield 등 외부 AI 생성 툴에 프롬프트를 붙여넣고, 프레임 이미지를 참조 이미지로 첨부

## 2. 카메라 조작 (인물 중심 궤도 회전)

ShotStage의 카메라는 **피사체를 중심(피벗)에 두고 카메라가 그 주위를 도는 방식**입니다. 아무리 회전해도 피사체가 화면 밖으로 벗어나지 않습니다.

| 조작 | 동작 |
|---|---|
| 드래그 | 피사체를 중심에 고정한 채 카메라가 궤도 회전 |
| `Space` + 드래그 | 핸드 툴 — 화면 자체를 이동(팬). 스페이스바를 누르면 커서가 손 모양으로 바뀜 |
| 스크롤 | 줌 인/아웃 (카메라-피사체 거리 조절) |
| `W` `A` `S` `D` | 회전 중심점을 수평으로 미세 이동 |
| `Q` / `E` | 회전 중심점의 높이 조절 |

- **피사체 중심으로 재정렬** 버튼: 드래그로 회전했거나 WASD/스페이스로 중심점을 옮긴 것을 모두 현재 샷 타입 기준 프레이밍으로 되돌립니다.
- **Bird's-eye** 전환: 위에서 내려다보는 시점. 이 모드에서는 드래그가 화면 이동, 스크롤이 고도(줌) 조절로 동작합니다. 샷 타입을 다시 선택하면 인물 위치로 중심이 리셋됩니다.

## 3. 샷 타입

| 버튼 | 설명 |
|---|---|
| 미디엄 샷 | 허리~머리 위 정도가 보이는 표준 샷 |
| 와이드 샷 | 전신과 주변 공간이 넓게 보이는 샷 |
| 클로즈업 | 얼굴에 초점을 맞춘 샷 |
| 로우 앵글 | 아래에서 올려다보는 앵글 |
| 하이 앵글 | 위에서 내려다보는 앵글 |
| 오버 숄더 | 두 번째 인물의 어깨 너머로 첫 번째 인물을 보는 대화 구도 |
| 투 샷 | 두 인물이 함께 나오는 구도 |

오버 숄더 / 투 샷을 선택하면 두 번째 피사체가 자동으로 함께 나타납니다.

## 4. 카메라 패널

- **Lens (화각)**: 슬라이더 또는 12 / 15 / 21 / 24 / 35 / 50 / 85 / 135mm 프리셋 버튼
- **화각비**: 16:9, 9:16, 1:1, 4:5, 2.35:1 중 선택. 뷰포트에 연한 크롭 가이드라인이 표시되고, **실제 생성되는 프레임 이미지도 그 비율로 잘려서 저장**됩니다 (화면에 보이는 크롭과 실제 다운로드 이미지가 항상 일치).

## 5. 피사체 패널

- **인물 1**: 좌우 / 깊이 / 회전 슬라이더로 위치 조정
- **두 번째 피사체 표시**를 켜면 **인물 2** 슬라이더가 별도로 나타나며, 인물 1과 완전히 독립적으로 조정할 수 있습니다.
- 인물 1과 인물 2는 3D 모델 색과 패널 슬라이더 색이 서로 같아서, 어떤 슬라이더가 어떤 인물을 조정하는지 바로 구분됩니다.

## 6. 프롬프트 패널

- **Image / Video 모드** 전환
- **모델 선택**: Image 모드는 GPT Image 2 / Nano Banana Pro / Midjourney / Flux, Video 모드는 Higgsfield / Seedance 2.0 / Runway / Kling 중 선택. 모델에 따라 프롬프트 형식이 최적화되어 생성됩니다 (예: Midjourney는 `--ar`, `--v 8.1` 파라미터를 문장 끝에 자동 추가, Seedance 2.0은 "누가 어떤 행동을 하는지"를 먼저 명시하고 참조 자료를 자연어로 지정하는 방식).
- **캐릭터시트 / 환경시트**: 체크박스를 켜면 이미지 업로드 영역이 나타나고, 업로드하면 "참조 이미지를 활용하라"는 문구가 프롬프트에 자동으로 추가됩니다. 체크박스 옆 `?` 아이콘에 마우스를 올리면 설명이 나옵니다.
- **두 번째 피사체(인물 2)가 있을 때**: "인물 2 (피사체)" 입력창과 "인물 2 캐릭터시트" 체크박스가 별도로 나타나서, 두 인물의 외형·참조 이미지를 각각 독립적으로 지정할 수 있습니다.
- **Subject / Environment / Look·style 입력창**: 옆의 별 아이콘으로 자주 쓰는 설정을 저장 · 불러오기 할 수 있습니다 (브라우저 로컬에 저장, 다른 기기와는 동기화되지 않음).
- **Video 모드 전용**: 카메라 무브(Pan / Tilt / Dolly / Track / Arc / Handheld) 선택, 강도(약함/보통/강함, 마우스 오버 시 설명 표시), 지속 시간(초) 지정 가능.

## 7. 결과 모달

- 블로킹 프레임 미리보기 + 완성된 프롬프트 텍스트
- **프롬프트 복사**, **프레임 다운로드** 버튼
- **Higgsfield에서 만들기** 버튼으로 바로 이동 가능

## 8. 알려진 제한 사항

- 캐릭터·환경·스타일 프리셋은 브라우저 로컬(IndexedDB)에만 저장됩니다. 다른 기기/브라우저와 동기화되지 않습니다.
- 업로드한 참조 이미지는 리사이즈 없이 원본 그대로 저장됩니다. 매우 큰 이미지를 많이 저장하면 브라우저 저장 용량 제한에 걸릴 수 있습니다.
- AI 모델 드롭다운은 프롬프트 형식만 최적화할 뿐, 실제 생성 API 연동은 하지 않습니다 (프롬프트 텍스트 생성까지가 이 도구의 역할).
- 마네킹 관절은 향후 "관절을 움직이는" 기능을 위해 부모-자식 구조로 준비되어 있지만, 현재는 포즈를 직접 바꾸는 UI는 아직 없습니다.
