# PRD — ShotStage (고도화 버전)

**문서 버전**: v0.1 (초안)
**작성일**: 2026-07-15
**작성 목적**: 원본 ShotStage(https://shotstage.vercel.app/)의 핵심 기능을 유지하면서, 카메라 고급 기능과 캐릭터·환경 에셋 라이브러리를 추가한 개인 크리에이터용 고도화 버전 개발
**배포 계획**: GitHub 저장소 → Vercel 배포

---

## 1. 배경

ShotStage는 3D 뷰포트에서 마네킹(스탠드인 캐릭터)을 배치하고 카메라 앵글·렌즈·샷 타입을 정확히 설정한 뒤, 그 설정을 그대로 자연어 프롬프트와 참조 이미지(블로킹 프레임)로 변환해주는 도구다. 사용자는 이 결과물을 GPT Image 2, Higgsfield 같은 외부 AI 생성 툴에 붙여넣어 원하는 샷을 정확히 재현할 수 있다.

핵심 가치는 "카메라 언어를 정확한 3D 프레이밍으로 확정하고, 그걸 그대로 생성형 AI가 이해할 수 있는 프롬프트로 변환한다"는 데 있다. 이번 고도화는 이 핵심 가치를 해치지 않는 선에서, 다음 두 축을 확장한다.

1. **카메라 고급 기능** — 정지된 한 컷뿐 아니라 무빙샷(비디오 카메라워크)과 화각비까지 다룰 수 있게 한다.
2. **캐릭터·환경 에셋 라이브러리** — 매번 텍스트를 새로 입력하지 않고, 자주 쓰는 캐릭터·환경 설정을 저장하고 재사용할 수 있게 한다.

## 2. 목표

| 목표 | 설명 |
|---|---|
| G1. 반복 입력 제거 | 같은 캐릭터/환경을 여러 샷에서 쓸 때 매번 텍스트를 새로 치지 않아도 된다 |
| G2. 비디오 프롬프트 품질 향상 | 정지 샷뿐 아니라 카메라 무빙이 있는 비디오 생성 프롬프트도 정밀하게 뽑을 수 있다 |
| G3. 기존 워크플로우 유지 | "3D로 프레이밍 → 프롬프트 생성 → 외부 AI 툴에 붙여넣기"라는 원래 흐름은 그대로 유지한다 |

**비목표(Non-goal)**: 이번 범위에서는 팀 협업, 계정/로그인, 서버 저장을 다루지 않는다 (3절 참고).

## 3. 타겟 사용자 & 범위

- **타겟 사용자**: 개인 크리에이터, 1인 작업 중심. 여러 사람이 프로젝트를 공유하거나 코멘트를 남기는 협업 시나리오는 이번 범위에 포함하지 않는다.
- **PRD 범위**: MVP 고도화. 아래 두 기능 영역만 다루며, 그 외 확장(멀티 샷 스토리보드, AI 모델 연동 확장 등)은 "8. 향후 고려 사항"에 아이디어만 남겨둔다.
- **저장 방식**: 계정/서버가 없는 개인용 도구이므로, 브라우저 로컬 저장(IndexedDB)만으로 에셋 라이브러리를 구현한다. 이는 이번 PRD의 핵심 가정이며, 만약 여러 기기 간 동기화가 필요하다면 별도 논의가 필요하다.

## 4. 기존 기능 (변경하지 않음)

아래는 원본 ShotStage의 기존 기능이며, 이번 고도화에서 그대로 유지한다.

- 3D 뷰포트: WASD 이동, 드래그 시점 회전, 스크롤 돌리, Q/E 상하 이동
- Shot view / Bird's-eye 뷰 전환
- 샷 타입 프리셋: Medium, Wide, Close-Up, Low Angle, High Angle, Over Shoulder, Two Shot
- 카메라 Lens(FOV) 슬라이더, Recenter on subject
- 피사체 좌우/깊이/회전 슬라이더, 두 번째 피사체 표시
- 프롬프트 패널: Image/Video 모드 토글, AI 모델 선택, 캐릭터시트·환경시트 보유 체크박스, Subject/Environment/Look-style 입력
- 결과 모달: 블로킹 프레임 다운로드, 프롬프트 복사, Higgsfield 연동 버튼

## 5. 신규 기능 A — 카메라 고급 기능

### 5.1 화각비(Aspect Ratio) 선택
- 프롬프트 패널에 화각비 선택 UI 추가: `16:9`, `9:16`, `1:1`, `4:5`, `2.35:1`
- 3D 뷰포트 자체도 선택된 화각비에 맞춰 프레임 마스크를 표시한다 (현재는 고정 16:9로 추정).
- 생성되는 프롬프트 텍스트의 마지막 화각비 표기(`16:9` 부분)가 선택값으로 대체된다.

### 5.2 렌즈 프리셋
- 현재 FOV 슬라이더(도 단위)는 유지하되, 실제 촬영에서 익숙한 mm 단위 프리셋 버튼을 함께 제공한다: `24mm`, `35mm`, `50mm`, `85mm`, `135mm`
- 프리셋 클릭 시 슬라이더 값이 해당 mm에 대응하는 FOV로 스냅되고, 표시 라벨도 mm 단위로 통일한다 (현재 UI에 이미 `28mm` 표기가 있으므로, 내부적으로 FOV(도) ↔ mm 환산 로직이 이미 있거나 필요하다는 가정).

### 5.3 무빙샷 / 비디오 카메라워크
Video 모드를 선택했을 때만 노출되는 신규 섹션이다. 기존 원본과 동일하게 **프레임은 한 장만 확정**하는 흐름을 그대로 유지하고, 거기에 카메라 무브 정보만 얹는다.

- 지금처럼 카메라를 움직여 프레임 하나를 확정한다 (변경 없음)
- **카메라 무브 타입 선택** (프리셋 버튼, 단일 선택): `Static`(고정), `Pan Left/Right`(패닝), `Tilt Up/Down`(틸팅), `Dolly In/Out`(달리), `Track Left/Right`(트래킹), `Arc`(원호 이동), `Handheld`(핸드헬드 흔들림)
- **무브 강도**: 약함/보통/강함 3단계 버튼. 강도를 미리 눈으로 확인할 방법이 없으므로, 각 버튼에 결과를 가늠할 수 있는 짧은 설명 툴팁을 붙인다.
  - 약함: "거의 느껴지지 않는 미세한 움직임"
  - 보통: "자연스럽게 체감되는 표준적인 움직임"
  - 강함: "뚜렷하고 극적인 움직임"
- **지속 시간 입력**: 초 단위 (예: 3초, 5초, 10초)
- 블로킹 프레임은 기존과 동일하게 1장만 캡처·다운로드한다 (시작/끝 2단계 캡처는 하지 않는다)
- 생성되는 프롬프트에 무브를 설명하는 한 문장이 추가된다. 예: `Starting from this medium shot, the camera slowly dollies in over 5 seconds, maintaining eye level, 35mm lens.`

## 6. 신규 기능 B — 캐릭터·환경 에셋 라이브러리

### 6.1 저장 대상
라이브러리는 세 종류의 프리셋을 저장한다.

| 프리셋 종류 | 저장 내용 |
|---|---|
| 캐릭터 프리셋 | 이름, Subject 텍스트(예: "a young woman in a tan coat"), 참조 이미지(선택, 캐릭터시트 이미지 업로드) |
| 환경 프리셋 | 이름, Environment 텍스트(예: "a sunlit modern living room"), 참조 이미지(선택, 환경시트 이미지 업로드) |
| 스타일 프리셋 | 이름, Look/style 텍스트(예: "moody cinematic lighting") |

### 6.2 UI 변경
- PROMPT 섹션의 Subject / Environment / Look-style 각 입력창 옆에 "라이브러리에서 불러오기" 아이콘 버튼을 추가한다
- 클릭 시 저장된 프리셋 목록이 드롭다운으로 표시되고, 선택하면 해당 텍스트(와 참조 이미지가 있다면 이미지)가 자동으로 채워진다
- 각 입력창 옆에는 "현재 값을 라이브러리에 저장" 버튼도 함께 제공한다. 이미 있는 이름으로 저장하면 덮어쓰기된다
- 별도의 관리 화면은 만들지 않는다. 드롭다운 목록의 각 항목에 마우스를 올리면 작은 삭제(×) 아이콘이 나타나고, 클릭하면 바로 삭제된다. 이름 변경은 "덮어쓰기 저장"으로 대신한다

### 6.3 캐릭터시트 · 환경시트 이미지 연동
- 기존 UI에는 "I have a character sheet" / "I have an environment sheet" 체크박스만 있고 실제 업로드 기능은 확인되지 않았다. 이번 고도화에서는 체크박스 선택 시 이미지 업로드 영역이 나타나도록 확장한다
- 업로드된 이미지는 캐릭터/환경 프리셋으로 저장할 때 함께 저장되어, 다음에 같은 프리셋을 불러오면 이미지도 함께 복원된다
- 프롬프트 생성 시, 업로드된 참조 이미지가 있다는 사실이 프롬프트 안내 문구에 반영된다 (예: "Use the attached character sheet to match the subject's appearance exactly")

### 6.4 데이터 모델 (IndexedDB, 로컬 저장)

```
Store: characterPresets
  - id: string (uuid)
  - name: string
  - subjectText: string
  - referenceImage: Blob | null
  - createdAt: number

Store: environmentPresets
  - id: string (uuid)
  - name: string
  - environmentText: string
  - referenceImage: Blob | null
  - createdAt: number

Store: stylePresets
  - id: string (uuid)
  - name: string
  - styleText: string
  - createdAt: number
```

## 7. 기술 스택 제안

| 영역 | 제안 |
|---|---|
| 프레임워크 | Next.js (App Router) |
| 3D 렌더링 | React Three Fiber + drei (기존 원본도 유사한 스택으로 추정) |
| 상태 관리 | Zustand (뷰포트 카메라 상태, 프롬프트 입력 상태) |
| 로컬 저장 | IndexedDB (idb 라이브러리 사용 권장) |
| 스타일링 | Tailwind CSS |
| 배포 | Vercel + GitHub 연동 (원본과 동일) |

이번 PRD 범위(로컬 저장만 사용)에서는 별도 백엔드/DB 서버, 인증 시스템이 필요 없다. 이는 개인 크리에이터 타겟과 MVP 범위라는 전제에 기반한 가정이므로, 추후 팀 협업 기능을 추가하게 되면 이 부분부터 재설계가 필요하다.

## 8. 향후 고려 사항 (이번 범위 밖, 참고용)

다음은 이번 PRD에는 포함하지 않지만, 사용자가 우선순위 선택에서 제외한 영역들이다. 추후 별도 PRD로 다룰 수 있다.

- 멀티 샷 / 스토리보드 관리 (씬 단위로 여러 샷을 저장하고 나열)
- AI 모델 연동 확장 (여러 모델 지원, 프롬프트 히스토리, 앱 내 직접 생성)
- 팀 협업 (공유 링크, 코멘트, 서버 저장/계정)

## 9. 오픈 이슈 / 확인 필요 사항

- 원본 사이트가 이미 mm ↔ FOV(도) 환산 로직을 갖고 있는지 확인이 필요하다 (현재는 UI상 두 단위가 혼용되어 있어 추정만 한 상태).
- 캐릭터/환경 참조 이미지를 IndexedDB에 저장할 때 용량 제한(브라우저별 상이)을 고려해, 이미지 리사이즈/압축 로직이 필요할 수 있다.

## 9-1. 자체 점검을 통해 걷어낸 항목 (기록용)

아래 세 가지는 최초 초안에 있었으나, 원본 사이트의 "단일 화면, 한 번에 하나 확정" 흐름과 어긋나고 개발 비용 대비 이득이 적어 이번 범위에서 제외했다.

- **심도(DOF) 실시간 블러 렌더링**: 3D 뷰포트는 원래 "placeholder이니 무시하라"고 프롬프트에 명시하는 설계라, 여기에 렌더링 비용을 들이는 실익이 적다. F-stop 값을 프롬프트 텍스트에 반영하는 것도 이번 범위에서는 넣지 않는다.
- **무빙샷 시작/끝 2단계 프레임 캡처**: 프레임을 2장 관리하는 완전히 새로운 워크플로우라 과했다. 대신 프레임 1장 + 무브 프리셋 선택 방식으로 대체했다 (5.3절).
- **에셋 라이브러리 별도 관리 화면**: 없던 "페이지" 개념을 추가하는 것이라 과했다. 대신 드롭다운 내 inline 삭제/덮어쓰기로 대체했다 (6.2절).
